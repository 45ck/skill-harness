# Support

Use GitHub issues for public questions, bug reports, and feature requests.

## Good Support Requests

Include:

- operating system and shell
- Go, Node.js, Python, and package manager versions when relevant
- the exact `skill-harness` command you ran
- whether the target repo is a single package or monorepo
- relevant output from `.skill-harness/setup-proof.json` when setup completed
- the checks you ran after the change

Do not include secrets, credentials, private logs, customer data, raw traces, or unredacted machine-local paths.

## Security Issues

Follow [SECURITY.md](SECURITY.md) instead of opening a public issue with vulnerability details.

## Maintainer Workflow

Maintainers use Beads for repo-local task tracking and may create or link Beads issues while triaging GitHub issues. Public contributors do not need Beads to participate.
