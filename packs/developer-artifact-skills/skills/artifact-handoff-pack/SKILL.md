# Artifact Handoff Pack

Use this skill when ending a session, handing work to another agent, or preparing a review bundle.

## Bundle Contents

- governing issue or task
- canonical artifact source
- generated review surface, if any
- manifest entry or freshness status, if generated review output exists
- changed files or relevant files
- verification evidence
- unresolved assumptions
- next executable action

## Rules

- Prefer links and exact paths over prose summaries.
- Keep generated HTML optional and source-backed.
- Do not hand off an HTML artifact without its canonical source.
- Do not hide failed or skipped verification.

## Output

### Governing Work
### Source Artifacts
### Review Surfaces
### Evidence
### Open Risks
