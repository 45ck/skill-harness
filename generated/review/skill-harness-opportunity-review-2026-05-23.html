Executive Read

The strongest opportunity is to make `skill-harness` a capability-profile system, not just a pack installer. It should understand when a repo wants desktop review artifacts, CLI-first source artifacts, media/demo tooling, governed agent execution, or a minimal setup.

The user's demo-machine idea is viable if it is scoped as structured post-capture editing: turn existing demo and QA evidence into polished silent cuts, slideshow reels, and review pages. Avoid positioning it as a general video editor.

Best First Slice

  • Add a documented `media` profile.
  • Incubate `packs/demo-production-skills`.
  • Start with `demo-social-cut`, `demo-review-surface`, and `qa-to-demo`.
  • Add install-proof artifacts and integration tests next.

Demo-Machine Video Skill

Recommended framing: a structured post-capture edit generator for demo-machine and manual-qa-machine artifacts. It should create polished silent cuts, slideshow reels, posters, and review bundles from evidence-backed runs.

Workflow

  • Start with a `.demo.yaml`, completed demo-machine run, or manual QA finding.
  • Read `events.json`, `quality.json`, storyboard, screenshot, and segment evidence.
  • Generate an edit plan with selected spans, still frames, transitions, title cards, and export targets.
  • Render silent short cuts or slideshow MP4s.
  • Generate a static review page comparing source spec, output media, and evidence.

Candidate Skills

  • `demo-social-cut`
  • `demo-slideshow-edit`
  • `demo-review-surface`
  • `qa-to-demo`
  • `demo-release-packager`

Constraint: do not make this a generic video editor. The differentiator is traceability from source flows and run evidence to polished outputs.

Risks

Profile Sprawl

Too many profiles will recreate flag complexity under better names. Start with six or fewer and make `default` boring.

Evidence Theater

Generated artifacts need provenance, freshness, and proof links. Pretty HTML without source and verification is not enough.

Global Mutation

Install and render commands mutate user-home directories. Integration tests need isolated temp homes before broadening the surface.

Next Decisions
