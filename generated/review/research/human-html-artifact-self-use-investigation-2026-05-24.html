Human HTML Artifact Self-Use Investigation

Why the prior discovery did not show a generated HTML review artifact, what the repo currently supports, and how skill-harness should self-use its own artifact doctrine.

Verdict

The repo is partially self-scaffolded. It has source-first policy, committed generated review surfaces, a manifest, and HTML safety gates. The missing piece is a generic non-model artifact renderer and an operating rule that human-facing discovery must create and surface an HTML review page.

Root cause: `scripts/generate-model-review.mjs` renders model artifacts only. Beads discovery work can complete without creating a `research`, `product`, `business`, `data`, or `ux` review surface.

Investigation Detail

Behavioral Gap

The prior response treated the request as discovery and Beads tracking. That closed the loop for agent task management, but not for a human review artifact.

  • No canonical research source was created.
  • No generated HTML review page was created.
  • No HTML artifact path was surfaced in the final response.

Technical Gap

The repo has a model review generator, but no generic visual-source renderer for research or planning artifacts.

  • Model artifacts render under `generated/review/models`.
  • Research artifacts have source directories only after this artifact.
  • Package scripts do not include a generic `artifacts:generate` command.
