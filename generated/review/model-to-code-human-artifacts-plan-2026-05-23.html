Model-To-Code Human Artifacts Plan

A source-backed plan for making canonical UML/C4/model artifacts map to code, generate Markdown for humans, and produce static browser review surfaces when the workflow calls for visual inspection.

Date: 2026-05-23 Issue: skill-harness-ra5 Source: docs/artifacts/source/model-to-code-human-artifacts-plan-2026-05-23.md Generated review surface

Executive Read

The practical target is not full round-trip UML. The target is a governed artifact loop where canonical text models map to source files, symbols, tests, specs, issues, and generated review evidence.

The highest-leverage first move is to self-scaffold `skill-harness` enough to use its own artifact contract, then add a computed suite graph and drift checks over packs, agents, loadouts, templates, and docs.

Why Browser Artifacts Were Missing

  • No root `.skill-harness/project.json`.
  • No root artifact manifest before this artifact.
  • No root artifact HTML policy checker.
  • No automatic local review command wired for this repo.

Artifact Loop

Recommended Architecture

1. Computed Suite Graph

Load existing canonical files and normalize packs, skills, agents, templates, artifact profiles, modelling modes, and setup outputs.

first slice No new giant catalog required.

2. Hybrid Trace Layer

`skill-harness` installs the artifact workflow; `agent-docs` / specgraph answers impact questions when available.

optional Basic checks still work without specgraph.

3. Target Repo Model Inventory

Scaffold context, domain, use-case, and update-matrix sources with stable IDs and mapping examples.

gap Existing setup creates folders but not enough durable starter content.

4. Mapping Contract

Require model IDs, code mappings, evidence links, update triggers, source hash, owner, and readiness status.

checkable Most fields fit the current manifest/checker direction.

5. Review Generator

Generate static HTML and optional generated Markdown dashboards from canonical model sources and manifest metadata.

needed The root repo does not currently have this script surface.

6. CI Gate

Fail on stale mappings, missing generated review pages, unsafe HTML, or unacknowledged model impact.

needed No root `.github/` workflow is present.

Implementation Slices

1. Self-ScaffoldAdd root project config, manifest, checks, model dirs.
2. Suite GraphAdd graph loader and deterministic doc generation.
3. Model InventoryScaffold starter context, domain, use-case, update matrix.
4. Mapping ChecksValidate paths, hashes, evidence, and review surfaces.
5. Browser WorkflowAdd review command or documented local preview path.
6. CI GatesRun tests, build, graph drift, artifact policy, smoke setup.

Optional seventh slice: extend `agent-docs` / specgraph with model nodes and impact edges so changed files can resolve to impacted models. Repos without specgraph should still keep the basic manifest, path, hash, and HTML checks.

Update Rules And Drift Verdicts

Change TypeRequired Model Action
User goal or workflow changesUpdate use-case and activity; maybe sequence.
Domain concepts or business rules changeUpdate domain; maybe state.
Service or component boundary changesUpdate context/container/component.
Deployment or trust boundary changesUpdate deployment and security evidence.
Internal refactor onlyRegenerate dependency evidence; durable UML usually unchanged.
Generated review is staleRegenerate from source or mark artifact not ready.
Drift VerdictMeaningLifecycle Mapping
`aligned`Source, mapping, evidence, and review surface match code/docs.`ready`
`source-missing`Behavior changed but no canonical model source exists.`needs-source`
`inventory-mismatch`Inventory, manifest, and touchpoints disagree.`needs-source`
`review-stale`Source or code changed after HTML generation.`stale`
`evidence-missing`Claims are under-supported.`needs-evidence`
`policy-failed`Manifest, model, or HTML policy checks fail.`unsafe`
`inconclusive`Impact cannot yet be determined.`inconclusive`

Next Move

Implement `skill-harness-wqp` first enough to self-scaffold this repo's artifact loop, then implement `skill-harness-wdv` as the reusable suite graph and drift check.

The browser artifact gap is both a workflow issue and a repo setup issue: agents should create generated review surfaces when asked, and the root repo should have the scaffolded manifest/check scripts that make this automatic.
